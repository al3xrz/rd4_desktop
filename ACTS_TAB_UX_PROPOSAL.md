# Acts tab UX proposal

Предложение по дизайну и UX вкладки актов в карточке договора.

## Главная задача вкладки

Вкладка актов должна быстро отвечать на вопросы:

```text
какие акты есть по договору
на какую сумму оказаны услуги
какие услуги входят в выбранный акт
что можно напечатать
```

Операторский сценарий:

```text
открыть договор -> создать акт -> добавить услуги -> напечатать акт или талоны -> вернуться к договору
```

## Структура вкладки

Рекомендуемая структура:

```text
Панель действий
Таблица актов
Сводка по выбранному акту
Печать и быстрые действия
```

Сейчас вкладка уже имеет две группы действий: `Акт` и `Печать`. Это понятная основа, её стоит сохранить.

## Панель действий

Основные действия:

```text
Создать акт
Открыть акт
Удалить акт
```

Действия печати:

```text
Талоны
Акт
Акт + Талоны
```

Рекомендации:

- `Создать акт` должно быть первым действием.
- `Открыть`, `Удалить`, печать должны быть disabled, если акт не выбран.
- После создания, изменения или удаления акта таблица должна обновляться, а баланс договора пересчитываться.
- После печати стоит показывать короткое сообщение в статусной строке или рядом с панелью.

## Таблица актов

Рекомендуемые колонки:

```text
Номер
Дата
Стоимость услуг
Комментарий
```

Выравнивание:

```text
Номер — по центру
Дата — по центру
Стоимость услуг — по правому краю
Комментарий — по левому краю
```

Денежные значения должны иметь одинаковый формат:

```text
1000.00
12500.50
```

Если позже появится больше данных, можно добавить:

```text
Кол-во услуг
Печать
Автор
Дата создания
```

## Строки услуг внутри акта

В диалоге акта таблица услуг должна быть главным рабочим элементом.

Рекомендуемые колонки:

```text
Код
Услуга
Ед.
Цена
Кол-во
Скидка
Итого
Комментарий
```

Выравнивание:

```text
Код, Ед., Кол-во — по центру
Цена, Скидка, Итого — по правому краю
Услуга, Комментарий — по левому краю
```

## Сводка

Под таблицей актов полезна компактная строка:

```text
Акт: A-001 | Услуг: 4 | Сумма: 12500.00 | Комментарий: ...
```

Если акт не выбран:

```text
Выберите акт
```

## Пустые состояния

Если актов нет:

```text
По договору ещё нет актов
Создайте акт, чтобы добавить оказанные услуги
```

Если акт выбран, но услуг нет:

```text
В акте нет услуг
Добавьте услугу из справочника
```

## Горячие клавиши

Полезный минимум:

```text
Enter — открыть выбранный акт
Delete — удалить выбранный акт после подтверждения
Ctrl+N — создать акт
Ctrl+P — печать акта
```

Для диалога акта:

```text
Insert или Ctrl+N — добавить услугу
Enter — изменить выбранную строку
Delete — удалить строку
Ctrl+S — сохранить акт
Esc — закрыть диалог
```

## Приоритет внедрения

1. Готово: единое горизонтальное выравнивание таблицы актов и таблицы услуг.
2. Готово: disabled-состояния кнопок, когда акт или строка услуги не выбраны.
3. Готово: строка сводки по выбранному акту.
4. Готово: горячие клавиши для вкладки актов и диалога акта.
5. Готово: пустое состояние для договора без актов.
6. Позже: добавить колонку `Кол-во услуг`, если это поможет операторам быстрее проверять акт.
