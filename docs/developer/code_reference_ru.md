# Назначение модулей и функций

Этот файл служит русскоязычным справочником по коду RD4 Desktop. Он описывает
назначение модулей, классов и функций без перегрузки исходников длинными
комментариями. Если меняется ответственность модуля или публичное поведение
метода, обновляйте этот справочник вместе с кодом.

## Точка входа

### `app/main.py`

- `main()` — консольная точка входа. Настраивает логирование, запускает миграции
  и передает управление desktop UI. Ошибки старта логируются и показываются
  пользователю отдельным критическим сообщением.
- `run_app()` — выполняет реальную последовательность запуска приложения:
  логирование, миграции, инициализация базы, начальный администратор и UI.
- `show_critical_startup_error()` — пытается показать ошибку старта через Qt.
  Если Qt недоступен, печатает текст в консоль.

## `app/core`

### `app/core/config.py`

- `Settings` — контейнер базовых настроек приложения.
- `Settings.database_url` — строит SQLAlchemy URL для локальной SQLite-базы.
- `load_settings()` — создает настройки из текущего окружения и путей данных.

### `app/core/database.py`

- `get_engine()` — лениво создает общий SQLAlchemy engine.
- `_register_sqlite_pragmas()` — подключает обязательные SQLite PRAGMA:
  foreign keys, WAL и synchronous=NORMAL.
- `set_sqlite_pragmas()` — callback SQLAlchemy, который применяет PRAGMA к
  каждому новому DBAPI-соединению.
- `get_session_factory()` — лениво создает фабрику SQLAlchemy-сессий.
- `create_session()` — возвращает новую сессию для service-level транзакции.
- `session_scope()` — context manager транзакции: commit при успехе, rollback
  при исключении, закрытие сессии в любом случае.
- `init_database()` — создает таблицы ORM, если они еще отсутствуют.

### `app/core/logging.py`

- `configure_logging()` — настраивает вывод логов в файл приложения и stderr.

### `app/core/migrations.py`

- `get_alembic_config()` — собирает Alembic config с актуальным URL базы.
- `run_migrations()` — применяет миграции до `head` при запуске приложения.

### `app/core/paths.py`

- `get_bundle_dir()` — определяет корень ресурсов для обычного запуска и
  PyInstaller-сборки.
- `get_data_dir()` — возвращает пользовательскую папку данных RD4.
- `ensure_data_dir()` — создает папку данных и возвращает путь к ней.
- `get_log_dir()` / `ensure_log_dir()` — возвращают или создают папку логов.
- `get_log_file()` — возвращает путь к основному файлу лога.
- `get_resource_path()` — строит путь к ресурсу приложения внутри исходников
  или сборки.

### `app/core/security.py`

- `hash_password()` — хеширует пароль через PBKDF2-HMAC-SHA256 с солью.
- `verify_password()` — проверяет пароль против сохраненного PBKDF2-хеша.

## `app/models`

ORM-модели описывают структуру таблиц и связи. В них не должно быть бизнес-логики
или обращений к сервисам.

- `user.py`: `Role` задает роли пользователей, `User` хранит учетные записи.
- `contract.py`: `Contract` хранит договор, данные пациентки, представителя,
  признаки платности/ФОМС, предоплаты и soft-delete.
- `payment.py`: `Payment` хранит оплаты, возвраты и сведения о распроведении.
- `act.py`: `Act` хранит акт по договору, номер, дату, автора и soft-delete.
- `act_med_service.py`: `ActMedService` хранит строку услуги в акте со снимком
  кода, названия, цены, скидки и количества.
- `med_service.py`: `MedService` хранит дерево папок и медицинских услуг.

## `app/repositories`

Репозитории — тонкий слой запросов SQLAlchemy. Они не коммитят транзакции и не
знают бизнес-правил.

### `base.py`

- `BaseRepository` — общий CRUD-помощник для ORM-моделей.
- `create()` — создает объект, добавляет его в сессию и делает `flush()`.
- `get()` — получает объект по первичному ключу с учетом soft-delete.
- `list()` — возвращает список по простым equality-фильтрам.
- `update()` — обновляет существующие атрибуты объекта и делает `flush()`.
- `soft_delete()` — ставит `deleted=True`, если поле есть; иначе удаляет физически.
- `_exclude_deleted()` — добавляет условие `deleted IS false`, когда модель
  поддерживает soft-delete.

### `user.py`

- `UserRepository` — запросы к пользователям.
- `get_by_username()` — ищет пользователя по логину.
- `list_active()` — возвращает активных пользователей для окна входа.

### `contract.py`

- `ContractRepository` — запросы к договорам и агрегированным итогам.
- `list()` — возвращает договоры без тяжелых связей платежей и актов.
- `get_by_number()` — ищет договор по номеру.
- `get_with_details()` — загружает договор вместе с платежами, актами и строками.
- `list_summaries()` — одним запросом считает начисления, оплаты, возвраты,
  баланс и статус по договорам.

### `payment.py`

- `PaymentRepository` — запросы к платежам.
- `list_for_contract()` — возвращает платежи договора.
- `list_financial_report_rows()` — выбирает проведенные платежи за период для
  финансового XLSX-отчета.

### `act.py`

- `ActRepository` — запросы к актам.
- `get_by_number()` — ищет акт по номеру.
- `get_with_services()` — загружает акт вместе со строками услуг.
- `list_for_contract()` — возвращает акты договора со строками.

### `act_med_service.py`

- `ActMedServiceRepository` — запросы к строкам услуг актов.
- `list_for_act()` — возвращает строки конкретного акта.
- `list_service_report_rows()` — выбирает строки услуг за период для отчетов.

### `med_service.py`

- `MedServiceRepository` — запросы к справочнику услуг.
- `list_children()` — возвращает дочерние элементы папки или корни дерева.
- `get_tree()` — возвращает корневые элементы дерева услуг.

## `app/services`

Сервисы владеют бизнес-операциями и границами транзакций. Один публичный метод,
который меняет бизнес-данные, соответствует одной service-level транзакции.

### `auth.py`

- `AuthService` — бизнес-логика пользователей и авторизации.
- `login()` — проверяет логин, активность пользователя и пароль.
- `list_login_users()` — возвращает активных пользователей для формы входа.
- `create_user()` — создает пользователя, проверяет уникальность логина и
  хеширует пароль.
- `change_password()` — меняет пароль после проверки старого пароля.
- `reset_password()` — администраторский сброс пароля.
- `list_users()` — список пользователей для администраторского экрана.
- `update_user()` — обновляет профиль пользователя, кроме пароля.
- `set_user_active()` — включает или отключает учетную запись.
- `_require_admin()` — централизованная проверка прав администратора.

### `contract.py`

- `ContractService` — операции с договорами и их финансовым состоянием.
- `create_contract()` — создает договор и стартовые платежи-предоплаты.
- `update_contract()` — обновляет договор, запрещая менять уже созданные
  предоплаты.
- `delete_contract()` — soft-delete договора, платежей, актов и строк услуг.
- `get_contract()` — возвращает договор с деталями.
- `list_contracts()` — возвращает список договоров по фильтрам.
- `list_contract_summaries()` — возвращает агрегированные итоги для реестра.
- `get_contract_summary()` — считает начисления, оплаты, возвраты, баланс и
  статус одного договора.
- `_prepayments_from_contract()` — извлекает предоплаты из данных договора.
- `_as_decimal()` — нормализует числа в `Decimal`.
- `_reject_changed_prepayment()` — запрещает менять поля предоплат после создания.

### `payment.py`

- `PaymentService` — операции оплат, возвратов и распроведения.
- `create_payment()` — создает положительную оплату.
- `create_refund()` — создает отрицательный возврат.
- `update_payment()` — редактирует проведенный платеж.
- `unpost_payment()` — распроводит платеж с причиной и пользователем.
- `list_payments()` — возвращает платежи договора.
- `_create()` — общая внутренняя ветка создания оплаты или возврата.

### `act.py`

- `ActService` — операции актов и строк медицинских услуг.
- `create_act()` — создает акт и строки услуг в одной транзакции; если номер
  пустой, генерирует `{номер договора} / {номер акта по договору}`.
- `update_act()` — обновляет реквизиты акта.
- `delete_act()` — soft-delete акта и его строк.
- `add_service()` — добавляет строку услуги в существующий акт.
- `update_service_row()` — редактирует цену, скидку, количество и комментарий
  строки, сохраняя снимок выбранной услуги неизменным.
- `remove_service_row()` — soft-delete строки услуги.
- `list_acts()` — список актов договора.
- `get_act()` — акт со строками.
- `list_service_rows()` — строки конкретного акта.
- `next_act_number()` — показывает следующий номер акта для UI-предзаполнения.
- `_build_service_row()` — собирает payload строки услуги со снимком справочника.
- `_next_act_number()` — строит следующий номер по количеству актов договора.
- `_add_or_increment_service_row()` — добавляет строку или увеличивает количество
  у совпадающей услуги с той же скидкой.
- `_find_matching_service_row()` — ищет строку для объединения.

### `med_service.py`

- `MedServiceService` — операции справочника услуг.
- `create_folder()` — создает папку справочника.
- `create_service()` — создает конечную медицинскую услугу.
- `update_med_service()` — обновляет папку или услугу с проверкой родителя.
- `delete_med_service()` — soft-delete элемента справочника.
- `get_med_service()` — получает элемент справочника.
- `get_tree()` — возвращает дерево услуг.
- `list_services()` — возвращает все элементы справочника.
- `list_folders()` — возвращает только папки.
- `_create()` — общая ветка создания папки или услуги.
- `_validate_parent()` — проверяет, что родитель существует, является папкой и
  не создает цикл.
- `_load_children()` — принудительно загружает дочерние элементы дерева.

### `docx.py`

- `DocxService` — рендер DOCX-документов по шаблонам.
- `render_paid_contract()` / `render_foms_contract()` — печать договора.
- `render_act_ticket()` — печать талонов акта.
- `render_act()` — печать акта по платному или ФОМС-шаблону.
- `open_document()` — открывает созданный DOCX системной программой.
- `_render_template()` — общий рендер `docxtpl` в temp-файл.
- `build_contract_context()` — контекст платного договора.
- `build_foms_contract_context()` — контекст договора ФОМС.
- `build_act_ticket_context()` — контекст талонов акта.
- `build_act_context()` — контекст печатного акта.
- `date_parts_ru()` / `date_ru()` — русское форматирование дат.
- `build_passport_text()` — собирает паспортную строку.
- `normalize_docx_text()` — нормализует текст для DOCX.
- `as_decimal()` — безопасно приводит значение к `Decimal`.
- `money_ru()` / `money_int()` — форматируют денежные значения.
- `money_with_words_ru()` / `money_words_ru()` — формируют сумму цифрами и
  прописью.
- `ruble_word()` — выбирает форму слова "рубль".
- `quantity_ru()` — форматирует количество.
- `percent_ru()` — форматирует процент скидки.
- `_number_words_ru()` — переводит целое число в русские слова.
- `triad_words()` — внутренняя функция `_number_words_ru`, формирует слова
  одной трехзначной группы.
- `plural()` — внутренняя функция `_number_words_ru`, выбирает форму слова.
- `_safe_filename()` — очищает имя выходного файла.

### `report.py`

- `ReportService` — формирование XLSX-отчетов.
- `render_services_report()` — отчет по услугам за период.
- `render_financial_report()` — финансовый отчет по платежам.
- `render_services_matrix_report()` — матрица услуг по дням: количество и стоимость.
- `open_report()` — открывает XLSX системной программой.
- `_load_services_report_rows()` — загружает строки услуг для отчета.
- `_load_financial_report_rows()` — загружает платежи для финансового отчета.
- `_load_services_matrix_data()` — загружает справочник услуг и строки актов.
- `_render_services_report_xlsx()` — пишет XLSX услуг.
- `_render_services_matrix_xlsx()` — пишет XLSX-матрицу.
- `_render_financial_report_xlsx()` — пишет финансовый XLSX.
- `_group_report_rows()` — группирует услуги по дате, договору, услуге и цене.
- `_services_with_report_rows()` — добавляет в матрицу услуги, которых уже нет
  в справочнике, но есть в исторических строках актов.
- `_build_services_matrix_metrics()` — агрегирует количество и стоимость по
  услуге и дате.
- `_discounted_price()` — считает цену с учетом скидки.
- `_iter_report_days()` — перебирает все дни периода, включая пустые.
- `_start_of_day()` — нормализует дату к началу дня.
- `_fill_services_matrix_sheet()` — заполняет один лист матрицы.
- `_matrix_value()` — приводит целые Decimal-значения к `int` для Excel.

### `bootstrap.py`, `smoke.py`, `exceptions.py`

- `ensure_initial_admin()` — создает начального администратора, если нет ни
  одного пользователя.
- `run_smoke()` — выполняет сквозной smoke-сценарий сервисов.
- `smoke.main()` — CLI-точка запуска smoke-сценария.
- `DomainError` — базовая ошибка домена для UI-обработки.
- `ValidationError` — ошибка пользовательского ввода.
- `NotFoundError` — сущность не найдена.
- `PermissionDeniedError` — недостаточно прав.
- `BusinessRuleError` — нарушение бизнес-правила.
- `DuplicateError` — конфликт уникальности.

## `app/ui`

UI-модули адаптируют сервисы к PySide2. Они не обращаются к базе напрямую.

### Табличные модели

- `contracts_table_model.py`: `ContractsTableModel` показывает реестр договоров,
  сортировочные значения, баланс, статус, цветовые признаки удаления/выписки.
- `acts_table_model.py`: `ActsTableModel` показывает акты договора и сумму услуг.
- `act_services_table_model.py`: `ActServicesTableModel` показывает строки услуг
  акта или pending-строки нового акта.
- `payments_table_model.py`: `PaymentsTableModel` показывает платежи, возвраты,
  статус проведения и причину распроведения.
- `users_table_model.py`: `UsersTableModel` показывает пользователей и визуально
  выделяет неактивные учетные записи.

Общие методы табличных моделей:

- `rowCount()` / `columnCount()` — сообщают Qt размер таблицы.
- `data()` — возвращает отображаемые значения, цвета, шрифты и сортировочные
  роли.
- `headerData()` — возвращает заголовки колонок и номера строк.
- `*_at()` — возвращает ORM-объект по строке модели.
- `set_*()` — заменяет данные модели через `beginResetModel()` / `endResetModel()`.
- вспомогательные `_format_*`, `_alignment`, `_status_*` — форматируют значения
  и роли Qt для конкретных колонок.

### Основные окна и страницы

- `application.py`: `run_ui()` запускает `QApplication`, окно входа и главное
  окно; `open_main_window()` открывает приложение после успешного логина.
- `login_window.py`: `LoginWindow` отвечает за выбор пользователя, ввод пароля,
  показ/скрытие пароля и вызов `AuthService.login()`.
- `main_window.py`: `MainWindow` собирает страницы, меню, отчеты и навигацию;
  `ReportPeriodDialog` выбирает период отчетов.
- `contracts_page.py`: `ContractsPage` показывает реестр договоров, фильтры,
  создание, клонирование, редактирование, печать и удаление договора.
- `contract_details_page.py`: `ContractDetailsPage` показывает карточку договора,
  сводку баланса, вкладки платежей и актов, редактирование и печать договора.
- `users_page.py`: `UsersPage` управляет пользователями: создание, редактирование,
  сброс пароля, активация, фильтр и контекстное меню.
- `med_services_page.py`: `MedServicesPage` управляет деревом справочника услуг:
  папки, услуги, редактирование, удаление, фильтр и контекстное меню.
- `acts_panel.py`: `ActsPanel` показывает акты договора, итоги, создание,
  открытие, удаление и печать актов/талонов.
- `payments_panel.py`: `PaymentsPanel` показывает платежи договора, итоги,
  оплату, возврат, редактирование и распроведение.

### Диалоги

- `contract_dialog.py`: `ContractDialog` создает и редактирует договор, управляет
  вкладками данных пациента/представителя, предоплатами, ФОМС, валидацией и
  подсветкой обязательных полей.
- `act_dialog.py`: `ActDialog` создает и редактирует акт, подставляет следующий
  номер, хранит pending-строки до сохранения, показывает итог и управляет строками.
- `act_service_row_dialog.py`: `ActServiceRowDialog` редактирует цену, количество,
  скидку и комментарий строки услуги.
- `med_service_picker_dialog.py`: `MedServicePickerDialog` выбирает конечную услугу
  из дерева справочника, фильтрует по тексту и запрещает выбирать папки.
- `med_service_dialog.py`: `MedServiceDialog` создает и редактирует папку или услугу
  справочника.
- `payment_dialog.py`: `PaymentDialog` вводит дату, сумму и комментарий оплаты
  или возврата.
- `unpost_payment_dialog.py`: `UnpostPaymentDialog` запрашивает причину
  распроведения платежа.
- `password_dialog.py`: `PasswordDialog` запрашивает новый пароль.
- `user_dialog.py`: `UserDialog` создает и редактирует пользователя, роль,
  активность, пароль и комментарий.

### UI-помощники

- `qt.py` — единая точка импорта PySide2-классов, чтобы остальные модули не
  дублировали длинные списки импортов.
- `icons.py` — назначает стандартные и кастомные иконки кнопкам, диалогам и меню.
- `toolbars.py` — создает единый стиль toolbar и toolbar-кнопок.
