# RD4 documentation

Документация проекта разделена по аудитории:

- [Пользовательская документация](user/README.md) — ежедневная работа в приложении.
- [Документация администратора](admin/README.md) — установка, данные, база, сборка и обслуживание.
- [Документация разработчика](developer/README.md) — архитектура, окружение, проверки и правила изменений.

## Правило синхронизации

После изменений в проекте нужно обновлять соответствующий раздел:

- UI и рабочие сценарии — `docs/user/`.
- установка, база данных, сборка, пути данных — `docs/admin/`.
- архитектура, зависимости, миграции, структура модулей, правила разработки — `docs/developer/`.

Если изменение затрагивает несколько аудиторий, обновляются все связанные разделы.
